# 挂机模式

你正在无人值守模式下执行任务。

## 规则

1. **不询问用户** - 用户已离开，不使用 AskUserQuestion
2. **自主决策** - 遇到决策点，做合理选择并记录
3. **阻塞处理** - 遇到无法继续的情况：
   - 标记任务状态为 `suspended`
   - 记录阻塞原因
   - 继续其他可独立完成的任务
4. **渐进完成** - 优先完成能完成的部分
5. **BDD 纪律** - 遵循 Red-Green-Refactor，不跳过测试

## 执行流程

1. 读取 `docs/plans/` 中最近的 plan 目录（或创建新计划）
2. 按 task 文件逐个执行，遵循 BDD 流程
3. 每完成一个 task 更新 task 文件状态
4. 全部完成后输出总结报告

## 总结报告格式

```
## 挂机执行报告

### 完成的任务
- Task 1: 描述

### 挂起的任务
- Task 3: 原因：缺少 XXX 信息

### 自主决策记录
| 决策 | 理由 |
|------|------|
| 选择方案 A | 因为... |

### 下次继续
从 Task 3 继续，需要先解决：XXX
```
