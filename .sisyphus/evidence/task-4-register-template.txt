## Task 4: Retained Exception Registration Template
Date: Fri Apr 10 2026
Plan: cloudflare-official-alignment

### Purpose

This file provides the canonical template for registering a Cloudflare deviation that
cannot be retired in the current wave. Wave 2+ tasks that fail to retire a deviation
MUST register it here before closing their task.

A retained exception is NOT a failure. It is a documented decision that this deviation
is still load-bearing and cannot be safely removed with currently available proof.

### Usage Instructions

1. Copy the template block below (between the --- markers).
2. Fill in every field. Do not leave any field blank.
3. Assign a stable exception-id using the format: CF-EXCEPT-NNN (e.g., CF-EXCEPT-001).
4. Reference this ID in any related build scripts, smoke scripts, or AGENTS.md entries.
5. When the retirement criteria are eventually met, create a new failed-retirement or
   successful-retirement record and update the status field to RETIRED with date.
6. The exception register lives at:
   docs/guides/CLOUDFLARE-RETAINED-EXCEPTION-REGISTER.md
   (to be created by Task 12 using entries from this template)

### Classification Guide

Type options:
  patch           - A monkey-patch applied to generated output after build
  wrapper         - A script that wraps an official CLI command to add behavior
  build-script    - A custom build script replacing or extending an official step
  deploy-script   - A custom deploy script replacing or extending an official step
  exception-contract - A checker/assertion script that gates build/deploy on a condition
  checker         - A script that reads generated artifacts and asserts invariants
  config          - An open-next.config.ts or wrangler config deviation from stock defaults

Issue category (from CLOUDFLARE-ISSUE-TAXONOMY.md):
  platform-entry        - Wrangler / Miniflare / preview startup / worker boot
  generated-artifact    - Manifest load, dynamic require, preview-only 500s
  runtime-regression    - Wrong redirects, cookies, headers, contact behavior
  deployment-behavior   - Deployed environment fails after passing local proof

Priority:
  high    - Retiring this is a prerequisite for meaningful official-alignment progress
  medium  - Retiring this would meaningfully reduce deviation surface but is not blocking
  low     - Retiring this is a housekeeping improvement only

Status:
  ACTIVE    - Currently load-bearing; must not be removed
  DEFERRED  - Retirement is planned but blocked on a specific external condition
  RETIRED   - Successfully retired (record the date and proof used)

accidentally re-attempting an already-documented retirement without reading this register.
