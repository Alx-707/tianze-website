#1  scripts/cloudflare/patch-prefetch-hints-manifest.mjs
────────────────────────────────────────────────────────────────────────────────
Type: WORKAROUND + EXCEPTION-CONTRACT

Purpose:
  After opennextjs-cloudflare build, the generated default worker artifact
  contains getMiddlewareManifest() implemented as a dynamic require().
  Cloudflare Workers do not support dynamic require() at runtime, so this
  script rewrites the generated artifact to use loadManifest() (static import
  equivalent) instead.

Five patch points (Fix A parts 1 & 2, Fix B part 1, Fix B part 2, Fix C):

  Fix A part 1 — requireManifestExternal declaration:
    Adds: const { loadManifest } = require("@opentelemetry/core")
    Note: The actual loadManifest path is the real OpenNext manifest loader,
          not opentelemetry — the comment in code uses a placeholder name;
          the real target is the generated _loadmanifestexternal chunk.

  Fix A part 2 — HARDCODED ABSOLUTE PATH BUG:
    Regex: /require\("\/Users\/Data\/Warehouse\/Pipe\/tianze-website\/\.open-next\/server-functions\/default\/\.next\/server\/app\/api\/([^"]+)"\)/g
    This regex contains the absolute filesystem path of the original developer's
    machine. This will silently fail (no match, no patch) on any machine where
    the repo is not located at exactly /Users/Data/Warehouse/Pipe/tianze-website/.
    This is a known defect. The replacement should use a path relative to
    process.cwd() or __dirname. Until fixed, CI and other developer machines
    may silently skip this patch point.

  Fix B part 1 — getMiddlewareManifest function body rewrite:
    Replaces the dynamic require() implementation of getMiddlewareManifest()
    with a call to loadManifest() using a relative path.

  Fix B part 2 — manifest path constant:
    Patches the string literal for middlewareManifestPath to use the correct
    relative form rather than the absolute path that Next.js generates.

  Fix C — prefetch hints route handler:
    Patches a secondary code path in the prefetch hints route handler that
    also calls getMiddlewareManifest() via dynamic require.

Opt-in gate:
  CF_APPLY_GENERATED_PATCH=true must be set to activate patching.
  Without it, the script runs in dry-run mode and exits with sentinel
  code 12 (DRY_RUN_REQUIRED_EXIT_CODE) if patching is needed.

Exit criteria (from generated-artifact-exception-contract.mjs):
  1. OpenNext generates the artifact without dynamic require() for middleware
     manifest.
  2. Cloudflare Workers natively support the require() call pattern used.
  3. The route is moved to a different worker that supports the pattern.
  4. The route is removed from the codebase entirely.

Critical known defect: The absolute path in Fix A part 2 makes this script
non-portable to other developer machines or CI environments that check out
the repo at a different path.

  - Runs opennextjs-cloudflare build (webpack mode, not turbopack)

  - Official equivalent: `opennextjs-cloudflare build` — but without the patch
    step, the generated artifact is broken for CF Workers runtime.

ADDITIONAL CROSS-CUTTING OBSERVATIONS
================================================================================

A. OPEN_NEXT_DEPLOY_ENV naming discrepancy:
   server-actions-secret-contract.mjs exports:
     OPEN_NEXT_DEPLOY_ENV = "OPEN_NEXT_DEPLOY"
   The export name is OPEN_NEXT_DEPLOY_ENV but the value (the env var name)
   is OPEN_NEXT_DEPLOY. deploy-phase6.mjs checks process.env.OPEN_NEXT_DEPLOY
   directly. These are consistent in behavior but the contract export name is
   confusing (sounds like the env var name includes _ENV). Minor risk of future
   misread.

B. PHASE6_ALIAS not drift-checked:
   check-alias-shim-contract.mjs only checks SINGLE_WORKER_ALIAS in wrangler.jsonc.
   PHASE6_ALIAS drift is only caught if build-phase6-workers.mjs correctly injects
   it — there is no separate contract checker for phase6-generated alias entries.

C. build-phase6-workers.mjs / phase-topology-contract.mjs drift history:
   AGENTS.md §24 documents a past incident where phase-topology-contract.mjs
   changed its internal API and build-phase6-workers.mjs was left on the old API.
   This class of drift is not protected by any current contract checker — it
   requires manual sync review whenever phase-topology-contract.mjs changes.

D. patch-prefetch-hints-manifest.mjs absolute path bug:
   The machine-specific path in Fix A part 2 is the single highest-portability
   risk in the entire deviation inventory. Silent skip (no error, just no patch)
   on non-matching machines makes it dangerous on CI if CI checkout path differs.

E. wrangler.jsonc minify: true vs open-next.config.ts minify: false:
   The interaction between these two settings is undocumented and needs
   clarification before any minification changes are attempted.
