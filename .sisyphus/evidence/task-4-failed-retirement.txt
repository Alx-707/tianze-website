## Task 4: Failed Retirement Documentation Template
Date: Fri Apr 10 2026
Plan: cloudflare-official-alignment

### Purpose

This file provides the canonical template for documenting a failed retirement attempt.
When a wave task tries to remove a Cloudflare deviation and cannot do so safely, the
result MUST be a documentation commit — not a force-removal commit.

### Core Principle

> Failed retirement attempts should produce documentation commits, not force-removal commits.

Force-removing a load-bearing deviation without proof is the most dangerous kind of
Cloudflare change in this repo. It converts a known deviation into a silent regression
that will only surface during deployed smoke or in production.

The correct outcome of a failed retirement:
  1. Understand WHY retirement is not yet safe.
  2. Document what was learned.
  3. Register the exception in the retained-exception register.
  4. Commit the documentation — not the broken removal.
  5. Leave the deviation in place, untouched.

### Usage Instructions

1. Copy the template block below (between the --- markers).
2. Fill in every field. Do not leave any field blank.
3. Link to the retained exception ID that was created as a result of this attempt
   (from task-4-register-template.txt format).
4. Commit this as a documentation commit with message:
   docs(cloudflare): record failed retirement of [exception-id]
5. Do NOT commit the incomplete removal code. Revert it first, then commit documentation.

### Proof Level Reference (from CLOUDFLARE-ISSUE-TAXONOMY.md)

Proof levels in ascending strength:
  L1: pnpm type-check / pnpm lint:check
  L2: pnpm build (Next.js standard build)
  L3: pnpm build:cf (Cloudflare Webpack build)
  L4: pnpm smoke:cf:preview (local page / redirect / cookie / header smoke)
  L4s: pnpm smoke:cf:preview:strict (adds /api/health; may fail on stock preview)
  L5: pnpm smoke:cf:deploy -- --base-url <url> (final deployed smoke)

Record which level was the first to fail. Do not claim higher-level proof if you
only reached a lower level before the failure.

### Commit Convention for Failed Retirement

When committing a failed retirement, use:

  git commit -m "docs(cloudflare): record failed retirement of [exception-id]"

Include in the commit:
  - This completed failed-retirement template file
  - The newly created or updated retained-exception register entry
  - Any AGENTS.md updates if new cautions were discovered

Do NOT include in the commit:
  - Any partially-removed deviation code
  - Any broken build artifacts
  - Any speculative "halfway fixed" state

The working state of the repo must be fully restored BEFORE creating the commit.
