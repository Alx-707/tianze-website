# Current Strategy Summary

这份文档把原 `ring1/`、`ring2/`、`ring3/` 的当前有效结论压缩成一份实现层可读摘要。
原 ring 分层文档已退出 live docs tree；需要追溯细节时看 git 历史或本机 Trash 批次。

## 1. 定位

Tianze 当前最有价值的差异化不是“产品目录更多”，而是：

> 精度和稳定性来自内部成型工艺、模具和生产控制。

对外表达要先讲买家结果，再讲背后原因，最后给证据：

1. 买家结果：精度、稳定性、工厂直供、定制响应。
2. 原因：内部掌握成型工艺、模具、打样和生产控制。
3. 证据：工艺流程、规格、公差、测试、样品、真实工厂资产。

不要把 Tianze 写成普通贸易商，也不要用“品质卓越”“行业领先”这类空话替代可验证事实。

## 2. 当前公开站结构

当前公开站先按已经上线的结构执行：

```text
/
├── /products/
│   ├── /products/north-america/
│   ├── /products/australia-new-zealand/
│   ├── /products/mexico/
│   ├── /products/europe/
│   └── /products/pneumatic-tube-systems/
├── /blog/
├── /about/
├── /contact/
├── /privacy/
└── /terms/
```

规则：

- 不要把未来目标路径误写成当前已上线 URL。
- OEM/定制能力只作为信息板块和询盘上下文存在，不再作为独立公开页面。
- Blog 当前恢复为最小资源列表页；它服务 SEO 支撑，不要喧宾夺主。

## 3. 内容策略

内容按买家决策顺序组织，不按公司自我介绍顺序组织。

优先级：

1. 首页分流：让买家快速理解 Tianze 和三类入口。
2. 产品/市场页：标准、规格、应用、样品和询价入口。
3. 首页/About/Contact 内的 OEM 能力说明：来图、模具、打样、质检、MOQ 和保密。
4. About / Contact：可信度与最终转化。
5. 法务、FAQ、下载材料：补足信任和上线完整性。

内容边界：

- 不让博客压过产品和询盘主线。
- 不把技术规格埋在大段宣传文案里。
- 能用表格、清单、对比讲清楚的，不写成长篇软文。

## 4. 多语言策略

英文站是当前主线：

- 服务出口买家。
- 对齐国际标准语境。
- 承担主要 SEO 与转化任务。

中文站保留基础版本，但不要求和英文站机械一比一镜像。
市场标准、合规语境、关键词和购买理由可以按语言/市场分开处理。

## 5. SEO 策略

SEO 结构要跟当前公开 URL 一致。当前重点不是铺很多词，而是用产品市场页和制造能力叙事建立可信度。

主线：

- 市场页和产品页优先。
- OEM/定制不再独立承接页面转化，统一引导到 Contact/询盘表单。
- 围绕内部成型工艺、模具、精度、过程控制、工厂直供等主题建立差异化。
- 不做空心关键词堆砌。

## 6. 信任系统

买家评估的是“这是不是一家可以放心合作的中国工厂”。

当前信任层级：

1. 第三方认证：强，但 Tianze 当前不宜只靠这条线和头部竞争。
2. 可验证制造事实：内部成型、模具、生产、质检。
3. 客户与市场证据：出口国家、复购、案例、反馈。
4. 过程透明：生产过程、QC、标准对照、工艺说明。
5. 低门槛承诺：样品、回复时效、工厂参观、认证配合。

不要用过量 badge 假装信任；能给过程和证据的地方，优先给过程和证据。

## 7. 转化路径

买家到站后，应在 2 次点击内进入 inquiry。

CTA 分层：

- 低承诺：查看认证状态、下载规格表。
- 中承诺：申请样品。
- 高承诺：获取报价、预约技术沟通、提交定制需求。

表单保持低摩擦，默认 4 个核心字段：

1. Name
2. Email
3. Product interest
4. Message

`Product interest` 应根据页面上下文预填，减少用户输入并保留来源信息。

## 8. 视觉与资产

视觉方向是：

> 工业制造的精度感 + 现代数字产品的克制感。

不是 Alibaba 展示页，也不是泛 SaaS 模板。

资产处理原则：

- 产品图：背景干净、构图统一、能看清尺寸和结构。
- 设备/工厂图：真实、整洁、有制造感，不像库存图。
- 认证文件：扫描清楚，页面展示版本和下载版本分开处理。
- 不过度修图，不把素材修成失真宣传图。

## 9. 实现层硬规则

- 路径改动必须同步 IA、内容、SEO 和代码。
- 文案、视觉、CTA 都要回到“买家结果 -> 背后原因 -> 可验证证据”。
- 当前规则、运行时真相、proof 口径仍以 `docs/guides/` 和 `docs/technical/` 为准。
