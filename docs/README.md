# Docs Overview

这个目录现在主要分成四类东西：

1. **主文档**：长期要看的总文档
2. **技术文档**：技术栈、缓存/部署等技术现状
3. **工作盘**：Superpowers / Impeccable / CWF 产物
4. **历史材料**：默认改看 git 历史或已移入 Trash 的批次，不再长期挂在 live docs tree

## 主文档

- `project-context.md`：项目背景、公司信息、业务信息
- `integrations.md`：后续可能引入的插件、组件、集成清单
- `design-truth.md`：当前设计真相文档

## 技术文档

位于 `technical/`：

- `technical/tech-stack.md`：纯技术栈信息
- `technical/next16-cache-notes.md`：Next.js 16 / Cache Components / i18n 缓存注意点
- `technical/deployment-notes.md`：Cloudflare / build:cf / preview / deploy 当前技术事实

## 治理合同

- `guides/`：当前真相、发布证明、治理合同

## 工作盘

- `superpowers/`：Superpowers 的 spec / plan / review 产物
- `impeccable/`：Impeccable / DWF 设计工作盘
- `cwf/`：CWF 文案工作盘与内容生成基础资料

## 历史材料

- 已退出当前真相面的旧文档，默认不再常驻 `docs/` 主树
- 需要回看时，优先用 git 历史；如本轮清理刚完成，也可从 Trash 批次恢复
